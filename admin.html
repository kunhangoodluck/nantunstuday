📖 方法一：Classic Token（推薦，最簡單）

  1. 前往 GitHub Classic Token 頁面
  2. Note 輸入名稱（例如「學習平台管理」）
  3. Expiration 選擇過期時間（建議 90 days 或 No expiration）
  4. 勾選 「repo」 權限（第一個選項，勾選後會自動全選子項目）
  5. 拉到最下面，點擊 「Generate token」
  6. 複製產生的 Token（以 ghp_ 開頭），貼到上方欄位

✅ Classic Token 只要勾選 repo 權限就能用，不容易出錯！

📖 方法二：Fine-grained Token（權限更精確）

  1. 前往 GitHub Fine-grained Token 頁面
  2. 點擊「Generate new token」
  3. 輸入名稱與過期時間
  4. Resource owner 確認是你自己的帳號
  5. Repository access 選「Only select repositories」→ 務必選到你的倉庫
  6. 展開 Permissions → Repository permissions：
    • Contents → 設為 Read and write
    • Metadata → 設為 Read-only（通常會自動勾選）
  7. 點擊「Generate token」並複製 Token

⚠️ 常見錯誤「Resource not accessible by personal access token」通常是因為：
　1. 沒有選到正確的倉庫（Repository access）
　2. Contents 權限沒有設為 Read and write（只有 Read 不夠）
　3. 如果倉庫屬於 Organization，需要 Organization 管理員核准 Token

🔒 Token 僅儲存在你的瀏覽器中（localStorage），不會傳送到任何第三方伺服器。
